# 📋 Contribution Checklist Templates

Quick checklists for different types of contributions.

---

## ✅ Lesson Improvement Checklist

- [ ] Content is clear for beginners
- [ ] Mathematical notation is correct
- [ ] Code examples are tested and work
- [ ] Examples use realistic/interesting data
- [ ] Explanations follow learning objectives
- [ ] Diagrams/visualizations are included where helpful
- [ ] Learning prerequisite marked clearly
- [ ] Follow-up resources/links included
- [ ] No typos or grammar errors
- [ ] Length is appropriate (not too long, covers topic)
- [ ] All cells can run in <5 minutes
- [ ] Notebook organized with clear section headings

---

## ✅ Exercise/Problem Set Checklist

- [ ] Problems are clearly stated
- [ ] Difficulty levels are varied (easy-medium-hard)
- [ ] Solutions provided separately or in solutions folder
- [ ] Solutions include explanations, not just answers
- [ ] All code solutions tested
- [ ] Solution code is clean and commented
- [ ] Hints provided (without giving away answers)
- [ ] Learning outcomes clear for each problem
- [ ] Mistakes to avoid are pointed out
- [ ] Bonus/stretch questions included
- [ ] Time estimate provided

---

## ✅ Documentation Checklist

- [ ] Topic is clearly explained
- [ ] Examples are provided
- [ ] Table of contents/navigation is clear
- [ ] Links to relevant lessons
- [ ] No dead links
- [ ] Markdown is properly formatted
- [ ] Code blocks are properly highlighted
- [ ] Screenshots/diagrams where helpful
- [ ] Search terms included (for discoverability)
- [ ] Updated main documentation index
- [ ] Proofread for typos

---

## ✅ Bug Fix Checklist

- [ ] Bug is clearly described
- [ ] Root cause identified
- [ ] Fix tested and verified
- [ ] No new bugs introduced
- [ ] Related documentation updated if needed
- [ ] Related tests updated/added
- [ ] Backward compatibility maintained
- [ ] Performance not negatively affected
- [ ] Error messages are clear

---

## ✅ Code Contribution Checklist

- [ ] Follows PEP 8 style guide
- [ ] Type hints included
- [ ] Docstrings complete (Google style)
- [ ] Comments explain WHY, not WHAT
- [ ] No hardcoded paths
- [ ] No sensitive data (API keys, passwords)
- [ ] Unit tests included
- [ ] Tests pass locally
- [ ] Code is DRY (Don't Repeat Yourself)
- [ ] No unused imports
- [ ] Performance considered
- [ ] Error handling included

---

## ✅ Banso Language/Cultural Content Checklist

- [ ] Orthography is correct (defer to native speakers)
- [ ] Cultural context is appropriate
- [ ] Sources cited
- [ ] Respectful representation of language
- [ ] Reviewed by native speakers (if possible)
- [ ] Pronunciation guidance included
- [ ] Modern usage notes included
- [ ] Related cultural information provided
- [ ] All translations verified
- [ ] Tone is respectful and educational

---

## ✅ Git/GitHub Checklist

- [ ] Forked from your account (not working on original)
- [ ] Feature branch created (not committing to main)
- [ ] Branch synced with upstream
- [ ] Commits follow conventional format
- [ ] No merge conflicts
- [ ] No debug code left in
- [ ] .gitignore rules respected
- [ ] No large files committed
- [ ] Commit history is clean
- [ ] PR title and description are complete

---

## ✅ PR Submission Checklist

- [ ] PR title matches convention (feat/fix/docs)
- [ ] PR description explains what and why
- [ ] All items in appropriate checklist above are met
- [ ] Related issue referenced (#123)
- [ ] Changes don't break existing functionality
- [ ] All new code has tests
- [ ] Documentation is updated
- [ ] No conflicts with main branch
- [ ] Ready for at least one reviewer to check
- [ ] Contact email included if not obvious

---

## ✅ Testing Checklist (for all PRs)

- [ ] Code runs without errors
- [ ] All cells/tests pass
- [ ] No deprecation warnings
- [ ] Works on Python 3.10+
- [ ] Works on Windows/Mac/Linux (if possible)
- [ ] Edge cases considered
- [ ] Performance acceptable
- [ ] Memory usage acceptable

---

## Quick Use

**Making a lesson improvement?** Use **Lesson Improvement Checklist**

**Adding practice problems?** Use **Exercise Checklist**

**Writing documentation?** Use **Documentation Checklist**

**Contributing code?** Use **Code Contribution** + **Testing** checklists

**Adding Banso content?** Use **Banso Language Checklist**

**Before pushing?** Use **Git Checklist** + **PR Submission Checklist**

---

All set? Great! You're ready to submit! 🚀
