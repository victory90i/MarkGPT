# Transfer Learning: Standing on Giants' Shoulders

## Why This Matters

Training a transformer from scratch requires 300B+ tokens and millions of dollars. Instead, **transfer learning** lets you:
1. Use pre-trained models (GPT-2, GPT-3, Llama, etc.)
2. Adapt them to new tasks with minimal data

This is how industry actually builds LLMs.

## Pre-training vs. Fine-tuning

### Pre-training (Corporate Scale)


## Full Fine-tuning (Simple but Expensive)

```python
model = load_pretrained_model("gpt2")  # 124M parameters

# Train on Bible data
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)

for epoch in range(3):
    for batch in dataloader:
        logits = model(batch['input_ids'])
        loss = compute_loss(logits, batch['labels'])
        
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
```

**Problem**: 
- 124M weights to update
- Huge memory (gradients + optimizer states)
- Slow training
- Risk of **catastrophic forgetting**: model forgets pre-trained knowledge

## LoRA: Parameter-Efficient Fine-tuning

### The Insight

Fine-tuning might not need to update *all* 124M weights. What if:
- Pretrained knowledge is in the "main" weights
- Task-specific knowledge fits in a small "adapter"?

**LoRA** (Hu et al., 2021): Instead of updating $W$ (dense matrix), update:

$$\Delta W = B \cdot A$$

Where $A$ is $(d, r)$ and $B$ is $(r, d)$ with $r \ll d$ (rank).

```
Full weight updates: d × d matrix → d² parameters
LoRA updates: (d×r) + (r×d) matrix → 2dr parameters

Example: d=4096, r=8
- Full: 4096² ≈ 16M parameters
- LoRA: 2 × 4096 × 8 ≈ 65K parameters
- **250x reduction!**
```

### Implementation

```python
class LoRALinear(nn.Module):
    """Replace Linear layer with LoRA adapter"""
    
    def __init__(self, in_features, out_features, r=8, alpha=16):
        super().__init__()
        
        # Pretrained weight (frozen)
        self.W = nn.Linear(in_features, out_features)
        self.W.weight.requires_grad_(False)
        
        # LoRA adapter
        self.A = nn.Linear(in_features, r, bias=False)      # Down-project
        self.B = nn.Linear(r, out_features, bias=False)     # Up-project
        
        # Scaling factor (usually alpha/r)
        self.scale = alpha / r
        
        # Initialize with small random weights
        nn.init.normal_(self.A.weight, 0, 0.02)
        nn.init.zeros_(self.B.weight)  # Start with zero delta
    
    def forward(self, x):
        # Original: W @ x
        out = self.W(x)
        
        # Add low-rank update
        out = out + self.scale * self.B(self.A(x))
        
        return out
```

### Training with LoRA

```python
model = load_pretrained_model("gpt2")

# Freeze all weights
for param in model.parameters():
    param.requires_grad = False

# Replace linear layers with LoRA adapters
for module in model.modules():
    if isinstance(module, nn.Linear):
        # In practice, only target attention layers
        # (full replacement would be overkill)
        pass

# Now only LoRA weights are trainable (~1-5% of model)
trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"Trainable: {trainable_params / 1e6:.1f}M out of 124M (1-2%)")

# Training is 10x faster, 5x less memory
optimizer = torch.optim.AdamW(
    [p for p in model.parameters() if p.requires_grad],
    lr=1e-3  # Higher learning rate for smaller updates
)

for batch in dataloader:
    logits = model(batch['input_ids'])
    loss = compute_loss(logits, batch['labels'])
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()  # Much faster!
```

### Merging LoRA for Inference

```python
# At inference, fold LoRA into weights
def merge_lora(model):
    """Merge LoRA weights into main weights (removes overhead)"""
    for module in model.modules():
        if isinstance(module, LoRALinear):
            # W_final = W_original + B @ A
            W_delta = module.B.weight @ module.A.weight
            module.W.weight.data += module.scale * W_delta
            
            # Mark as merged
            module.A.requires_grad_(False)
            module.B.requires_grad_(False)
```

## Multi-task Fine-tuning

Fine-tune on both English Bible + Banso proverbs:

```python
# Mix English and Banso data
train_loader_english = create_dataloader(bible_texts, batch_size=32)
train_loader_banso = create_dataloader(banso_texts, batch_size=32)

# Interleave batches
for english_batch, banso_batch in zip(train_loader_english, train_loader_banso):
    # Train on English
    loss_en = compute_loss(model(english_batch['input_ids']), english_batch['labels'])
    loss_en.backward()
    
    # Train on Banso (same model, different language)
    loss_banso = compute_loss(model(banso_batch['input_ids']), banso_batch['labels'])
    loss_banso.backward()
    
    optimizer.step()
    optimizer.zero_grad()

# Result: Model understands both languages!
```

## Key Takeaways

| Method | Speed | Memory | Performance |
|--------|-------|--------|-------------|
| No fine-tuning | 1x | 1x | Poor (generic model) |
| Full fine-tuning | 0.3x | 0.3x | Best (all parameters adapt) |
| LoRA | 1x | 0.7x | ~95% of full (much cheaper!) |
| Prompt engineering | 1x | 1x | Variable (fragile) |
