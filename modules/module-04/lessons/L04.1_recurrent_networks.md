# Recurrent Neural Networks

## Why This Matters

Before the Transformer era, RNNs were the dominant architecture for sequences. Understanding RNNs teaches you about **state**, **gradient flow through time**, and **sequence dependencies**—concepts that persist in modern architectures.

## The RNN Cell

An RNN processes sequences by maintaining a hidden state that updates at each timestep:

$$h_t = \sigma(W_{hh} h_{t-1} + W_{xh} x_t + b_h)$$
$$y_t = W_{hy} h_t + b_y$$

**Key**: The same weights $(W_{hh}, W_{xh}, W_{hy})$ apply at every timestep—this is how RNNs handle variable-length sequences.

## Implementing a Simple RNN

```python
import numpy as np

class SimpleRNN:
    """Basic RNN cell for sequence processing"""
    
    def __init__(self, input_size, hidden_size, output_size):
        self.Wxh = np.random.randn(input_size, hidden_size) * 0.01
        self.Whh = np.random.randn(hidden_size, hidden_size) * 0.01
        self.Why = np.random.randn(hidden_size, output_size) * 0.01
        self.bh = np.zeros(hidden_size)
        self.by = np.zeros(output_size)
    
    def forward(self, X_seq, h0=None):
        """Process sequence, return predictions and hidden states
        
        Args:
            X_seq: Shape (seq_len, batch_size, input_size)
            h0: Initial hidden state (batch_size, hidden_size)
            
        Returns:
            y_preds: Predictions at each timestep
            h_states: Hidden states (for backprop)
        """
        seq_len = X_seq.shape[0]
        batch_size = X_seq.shape[1]
        
        if h0 is None:
            h0 = np.zeros((batch_size, self.Whh.shape[0]))
        
        h = h0
        h_states = [h]
        y_preds = []
        
        for t in range(seq_len):
            x_t = X_seq[t]  # (batch_size, input_size)
            
            # RNN cell: h_t = tanh(Wxh @ x_t + Whh @ h_{t-1} + bh)
            h = np.tanh(x_t @ self.Wxh + h @ self.Whh + self.bh)
            
            # Output: y_t = h_t @ Why + by
            y_t = h @ self.Why + self.by
            
            h_states.append(h)
            y_preds.append(y_t)
        
        return y_preds, h_states
```

## The Vanishing Gradient Problem

When we backpropagate through many timesteps, gradients get multiplied by $\tanh'(z)$, which is typically < 1:

$$\frac{\partial L}{\partial W_{hh}} = \sum_{t=1}^{T} \frac{\partial L}{\partial y_t} \cdot \prod_{s=1}^{t} \tanh'(h_s)$$

If $\tanh'(h_s) \approx 0.1$, then after 50 timesteps, the gradient is $0.1^{50} \approx 10^{-50}$—essentially **zero**!

```python
# Vanilla gradient flow through 100 timesteps
grad_tanh = 0.1  # Typical tanh gradient
gradient = grad_tanh ** 100
print(f"Gradient magnitude: {gradient}")  # Essentially 0
```

**Result**: RNNs cannot learn long-range dependencies (> ~20 timesteps).

## Solution: LSTM (Long Short-Term Memory)

LSTMs use **gating mechanisms** to preserve gradient flow:

- **Forget gate**: $f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$ → How much to discard
- **Input gate**: $i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$ → How much new info to add
- **Output gate**: $o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$ → What to output

$$c_t = f_t \odot c_{t-1} + i_t \odot \tanh(W_c \cdot [h_{t-1}, x_t] + b_c)$$

The **cell state** $c_t$ has addition (not multiplication) → gradients don't vanish!

## Key Takeaway

- **RNNs**: Simple, intuitive, but can't handle long sequences
- **LSTM/GRU**: Add gating → preserve long-range dependencies
- **Transformers** (next module): Abandon recurrence entirely → even better!
