# Transformers: Building Modern LLMs

## Why This Matters

The Transformer architecture (2017) abandoned **recurrence** entirely—no more sequential processing! Instead, use **attention to connect all positions in parallel**. This single architectural decision enabled:
- Scaling to billions of parameters
- Training on massive datasets (entire internet)
- ChatGPT, Claude, Gemini, etc.

## The Transformer Block

```
Input Sequence
    ↓
[Multi-Head Attention] ← Query, Key, Value from same sequence
    ↓ (residual connection)
[Add & Normalize]
    ↓
[Feed-Forward Network] ← Two dense layers with ReLU
    ↓ (residual connection)
[Add & Normalize]
    ↓
Output (same shape as input)
```

## Decoder-Only Transformer (Like MarkGPT)

```python
import torch.nn as nn

class TransformerBlock(nn.Module):
    def __init__(self, d_model=512, num_heads=8, d_ff=2048):
        super().__init__()
        
        # Multi-head self-attention (token attends to all previous tokens)
        self.attention = MultiHeadAttention(d_model, num_heads)
        self.norm1 = nn.LayerNorm(d_model)
        
        # Feed-forward: expand then contract
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Linear(d_ff, d_model),
        )
        self.norm2 = nn.LayerNorm(d_model)
    
    def forward(self, x, causal_mask=None):
        # Pre-norm (normalize before operation, not after)
        # Why? Better gradient flow for very deep models
        
        # Attention path
        attended = self.attention(x, x, x, mask=causal_mask)
        x = x + attended  # Residual connection
        x = self.norm1(x)
        
        # FFN path
        ffn_out = self.ffn(x)
        x = x + ffn_out  # Residual connection
        x = self.norm2(x)
        
        return x
```

## Positional Encoding

Attention is **permutation invariant**: tokens [A, B, C] and [B, A, C] produce same attention pattern! We need to encode **position**.

### Learned Positional Embeddings

```python
# Simple: learnable parameter for each position
pos_embed = nn.Embedding(max_seq_len, d_model)  # (max_seq_len, d_model)
```

**Problem**: Model can't generalize to unseen positions.

### Positional Encoding (Vaswani et al., 2017)

Use **different frequencies** of sine/cosine waves:

$$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d_\text{model}}}\right)$$
$$PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d_\text{model}}}\right)$$

```python
import math

def positional_encoding(max_len, d_model):
    """Generate positional encodings for all positions"""
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len).unsqueeze(1)  # (max_len, 1)
    
    # Divide frequencies
    i = torch.arange(0, d_model, 2)
    div_term = 10000 ** (i / d_model)  # (d_model//2,)
    
    # Apply sin/cos
    pe[:, 0::2] = torch.sin(position / div_term)      # Even indices
    pe[:, 1::2] = torch.cos(position / div_term[:d_model//2])  # Odd indices
    
    return pe  # (max_len, d_model)
```

**Why this design?**
- Low frequencies (large wavelengths) vary slowly → capture global position
- High frequencies (small wavelengths) vary quickly → capture local position
- **Relative position encoding**: $PE(pos + k)$ can be expressed as linear function of $PE(pos)$

## Full Decoder-Only Model

```python
class DecoderOnlyTransformer(nn.Module):
    def __init__(self, vocab_size, d_model=512, num_heads=8, num_layers=6, d_ff=2048):
        super().__init__()
        
        # Embeddings
        self.token_embed = nn.Embedding(vocab_size, d_model)
        self.pos_embed = nn.Embedding(max_seq_len, d_model)  # Could be PositionalEncoding
        
        # Transformer blocks
        self.layers = nn.ModuleList([
            TransformerBlock(d_model, num_heads, d_ff)
            for _ in range(num_layers)
        ])
        
        # Language modeling head
        self.lm_head = nn.Linear(d_model, vocab_size)
    
    def forward(self, input_ids, targets=None):
        # Embed tokens
        embeds = self.token_embed(input_ids)  # (batch, seq_len, d_model)
        
        # Add positional encoding
        pos_ids = torch.arange(input_ids.shape[1]).unsqueeze(0)
        pos_embeds = self.pos_embed(pos_ids)
        x = embeds + pos_embeds
        
        # Create causal mask
        seq_len = input_ids.shape[1]
        causal_mask = torch.tril(torch.ones(seq_len, seq_len))
        
        # Pass through transformer layers
        for layer in self.layers:
            x = layer(x, causal_mask)
        
        # Project to vocabulary
        logits = self.lm_head(x)  # (batch, seq_len, vocab_size)
        
        # Compute loss if targets provided
        if targets is not None:
            loss = F.cross_entropy(
                logits.view(-1, vocab_size),
                targets.view(-1),
            )
            return logits, loss
        
        return logits
```

## Key Design Insights

| Aspect | Why |
|--------|-----|
| **Pre-norm** | Better gradient flow in deep models (50+ layers) |
| **Residual connections** | Prevent information bottleneck, enable very deep networks |
| **Multi-head attention** | Different representation subspaces capture different patterns |
| **Feed-forward (d_ff > d_model)** | Provide capacity for non-linear transformations |
| **Layer norm** | Stabilize training, prevent activations from exploding |

## MarkGPT's Architecture

```
Input: [0, 1, 5, 2, 3]  (tokenized Bible verse)
    ↓ Token Embedding (vocab_size → 128 or 256)
    ↓ Add Positional Embeddings
    ↓ [TransformerBlock] × num_layers (4 for Nano, 6 for Small)
       - Multi-head causal attention (8 heads)
       - Feed-forward (4× expansion)
    ↓ Language Model Head (d_model → vocab_size)
Output: logits for next token
```

## Scaling Laws

Empirical observation (Hoffmann et al., 2022): Optimal performance comes from scaling:

$$L(N, D) \approx E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$$

Where:
- $N$ = model size (parameters)
- $D$ = dataset size (tokens)
- Optimal allocation ≈ 20 tokens per parameter

**Example**: For a 10B parameter model, optimal training on ≈200B tokens.
