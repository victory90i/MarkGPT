# Attention: Fixing RNNs with Selective Focus

## Why This Matters

RNNs process sequences left-to-right, so they "forget" early tokens by the time they reach the end. **Attention** lets each position look back at *any* previous position—like having the model take notes on what's important. This simple idea revolutionized NLP.

## The Problem

Consider translating: "The **bank** of the river is green"

An RNN must:
1. Read "The bank of the river is green"
2. Encode everything into one vector
3. Decode while generating translation

But the model must compress all context into a single vector! By the time it outputs the translation of "bank," it might have forgotten it's a river bank, not a financial bank.

## The Solution: attention(query, key, value)

Instead of compressing to one vector, keep *all* timesteps and use **attention** to pick relevant ones:

```
Query: "What am I trying to decode right now?"
Key:   "What does each input timestep represent?"
Value: "What information should I use?"
```

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

## Step-by-Step Attention

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Args:
        Q: Query, shape (batch, seq_len, d_model)
        K, V: Key, Value, shape (batch, seq_len, d_model)
        mask: Optional mask (prevents attending to future tokens)
        
    Returns:
        output: Attention-weighted values
        attention_weights: Softmax matrix (for visualization)
    """
    # 1. Compute attention scores
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(K.shape[-1])
    # Shape: (batch, seq_len, seq_len)
    # scores[b, i, j] = how relevant is token j when decoding position i?
    
    # 2. Apply mask (optional: prevent looking forward)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    
    # 3. Softmax: convert to probability distribution
    attention_weights = F.softmax(scores, dim=-1)  # (batch, seq_len, seq_len)
    # attention_weights[b, i, j] = probability of attending to j from i
    
    # 4. Weight and sum values
    output = torch.matmul(attention_weights, V)  # (batch, seq_len, d_model)
    
    return output, attention_weights
```

## Multi-Head Attention

One attention mechanism isn't enough. Different "representation subspaces" help:

- Head 1: Attend to syntactic role (subject, verb, object)
- Head 2: Attend to semantic similarity
- Head 3: Attend to positional relationships
- Head 4: Attend to discourse structure

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        assert d_model % num_heads == 0
        
        self.num_heads = num_heads
        self.d_k = d_model // num_heads
        
        # Project Q, K, V to each subspace
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
    
    def forward(self, Q, K, V, mask=None):
        batch_size = Q.shape[0]
        
        # Project and reshape for multi-head
        Q = self.W_q(Q).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(K).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        
        # Apply attention on each head
        output, _ = scaled_dot_product_attention(Q, K, V, mask)
        
        # Concatenate heads
        output = output.transpose(1, 2).contiguous()
        output = output.view(batch_size, -1, self.d_model)
        
        # Final linear
        return self.W_o(output)
```

**Why scale by $\frac{1}{\sqrt{d_k}}$?**

When $d_k$ is large, dot products become huge → softmax outputs nearly 0 or 1 → **no gradients**. Scaling keeps dot products in a reasonable range.

```python
# Example: large dot products kill gradients
large_scores = torch.tensor([1000.0, 0.5, 0.1])
small_scores = torch.tensor([1.0, 0.5, 0.1])

print(F.softmax(large_scores))   # [1.0, 0.0, 0.0] - one token dominates
print(F.softmax(small_scores))   # [0.65, 0.24, 0.11] - distributed
```

## Attention in MarkGPT

In the MarkGPT model, attention is **causal** (can't attend to future tokens):

```python
# Create causal mask (upper triangular = future tokens)
mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1)
# Set future tokens to 0 (can't attend)
scores = scores.masked_fill(mask == 1, -1e9)
```

This ensures the model predicts each token using only:
- The token itself
- Previous tokens (past context)

**Not** future tokens (which haven't been generated yet).
