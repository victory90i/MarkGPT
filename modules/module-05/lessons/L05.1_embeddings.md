# Word Embeddings: Capturing Meaning in Vectors

## Why This Matters

Text is discrete (words are categories), but neural networks need continuous vectors. **Word embeddings** map discrete tokens to dense vectors where semantic relationships become geometric relationships. This is the foundation of all modern NLP.

## From One-Hot to Embeddings

### One-Hot Encoding (Bad)

```python
# Vocabulary: ["the", "bible", "speaks", "truth"]
# One-hot for "bible" (3 words don't know):
one_hot_bible = [0, 1, 0, 0]  # 75% zeros! Sparse!

# Problem: No semantic info. "bible" and "speaks" are equally different.
```

### Word Embeddings (Good)

```python
import np

# Same words, now as dense vectors:
embeddings = {
    "the": [0.02, 0.15, -0.18, 0.34],
    "bible": [0.71, 0.22, 0.18, 0.04],
    "speaks": [0.68, 0.25, 0.16, 0.12],
    "truth": [0.62, 0.30, 0.20, 0.08],
}

# Now compute similarities (dot product):
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(cosine_sim(embeddings["bible"], embeddings["truth"]))   # 0.97 (very similar!)
print(cosine_sim(embeddings["the"], embeddings["bible"]))     # 0.15 (different)
```

**Insight**: Words with similar meanings end up near each other in the embedding space!

## Word2Vec: Learning Embeddings

The key idea: **words are defined by the company they keep**.

```
Context: "The [?] speaks truth"

If [?] is frequently "bible": 
    → "bible" and "speaks" should be near each other
    → "bible" and "truth" should be near each other
```

**Word2Vec Skip-gram model**:

$$P(\text{context} | \text{target}) = \text{softmax}(\text{context}_\text{embed} \cdot \text{target}_\text{embed})$$

Given target word, predict its nearby words. This forces the embedding space to capture meaning!

```python
class SkipGramModel(nn.Module):
    """Word2Vec Skip-gram implementation"""
    
    def __init__(self, vocab_size, embedding_dim):
        super().__init__()
        # Target word embeddings: "what word are we predicting from?"
        self.target_embedding = nn.Embedding(vocab_size, embedding_dim)
        # Context word embeddings: "what word appears nearby?"
        self.context_embedding = nn.Embedding(vocab_size, embedding_dim)
    
    def forward(self, target_ids, context_ids):
        """
        Args:
            target_ids: Shape (batch,) - word we start with
            context_ids: Shape (batch,) - nearby word
            
        Returns:
            log probability the context appears near target
        """
        target_embed = self.target_embedding(target_ids)      # (batch, dim)
        context_embed = self.context_embedding(context_ids)    # (batch, dim)
        
        # Dot product: high if embeddings align
        logits = (target_embed * context_embed).sum(dim=1)     # (batch,)
        
        return logits
```

**Training**: Positive examples (real nearby words) vs. negative examples (random words). The model learns to distinguish them—and in doing so, captures meaning in the embeddings!

## GloVe: Combining Counting and Prediction

Word2Vec learns from prediction, but ignores **co-occurrence statistics** (how often words appear together in the corpus).

**GloVe** (Global Vectors) combines both:

$$J = \sum_{i,j} w(X_{ij}) \left( w_i^T w_j + b_i + b_j - \log X_{ij} \right)^2$$

Where:
- $X_{ij}$ = how often word $j$ appears in context of word $i$
- $w(X_{ij})$ = weight function to emphasize rare co-occurrences

**Why**: "bible" and "god" co-occur frequently (high $X$), so their embeddings should capture this. Skip-gram might miss this if the pair is rare in random windows.

For the Banso component, we need embeddings for both English and Lamnso':

## Exercise

Train Word2Vec on the King James Bible:

```python
from gensim.models import Word2Vec

# Load KJV text
texts = [text.split() for text in bible_verses]

# Train Word2Vec
model = Word2Vec(
    texts,
    vector_size=100,      # Embedding dimension
    window=5,             # Context window size
    min_count=5,          # Ignore rare words
    workers=4,            # Parallel training
)

# Explore learned embeddings
print(model.wv.most_similar("god"))       # Find similar words
print(model.wv.similarity("god", "lord"))  # Word similarity
```
